How retarded is this sorting algorithm and/or does it already exist?
It would only work in specific circumstances and requires a fair bit of mental but someone smarter than me could probably iron that out

An example with ints:
Find the minimum and maximum of an array. make a new array of length equal to the difference between them. Add the smallest value to each element such that they're all positive or 0. Put them in the new array with indices equal to their own values, and make every other value something outside of the range. Go through the long array, putting them back into the short array if they're within the range (ignoring the larger values that fill the space between them), subtract back the minimum value, e voila, it's sorted.

Drawbacks: probably only works with ints, repeat values would be difficult, range of original can only be half of integer max value, and many more I'm sure.
